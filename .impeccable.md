# Ping — Impeccable Design Context

## Design Context

### Users

**Who**: Close friends and intimate personal circles. Not work teams, not strangers — people who *matter* to each other.

**Context**: Used in moments of genuine connection — checking in on a friend, sharing a laugh, making up after an argument, celebrating something together. The interface is often one-handed, in bed, on a commute, or curled up on a sofa.

**Job to be done**: Send a message that *feels* like it came from a real person. Make the conversation itself feel like a ritual — not just a data transfer.

**Emotional goal**: When someone opens Ping, they should feel a quiet sense of warmth, like stepping into a familiar space that belongs to this relationship.

---

### Brand Personality

**Three words**: **溫暖 · 儀式感 · 親密** (Warm · Ritual · Intimate)

**Voice**: Understated confidence. Never shouts. Speaks softly but leaves an impression. Like a handwritten card vs. a text blast.

**Emotional register**: The interface should make small things feel special. A message sent is a moment. A Sonic Ping is a heartbeat. An apology is a gesture.

---

### Aesthetic Direction

**Visual tone**: macOS Glassmorphism — "Steel Frost" (dark) and "Kyoto Sunrise" (light). Not Discord's gamer darkness, not LINE's candy chaos. Closer to what Apple Messages would look like if it cared about emotional depth.

**Dark mode** (default): Deep steel gray (`oklch(0.18 0.005 240)`), frosted glass surfaces, Messages blue accent (`oklch(0.60 0.18 250)`). High contrast, low visual noise.

**Light mode** (secondary): Warm ivory white (`oklch(0.96 0.010 70)`), soft beige-gray cards, Japanese aesthetic warmth. Never pure white — always a hint of warmth.

**Color system**: OKLCH exclusively. Ritual accent colors drive emotional expression:
- **Sonic Ping / nudge**: Cyan (`--ritual-nudge`)
- **Apology**: Blue-purple (`oklch(0.60 0.20 270)`)
- **Celebration**: Warm gold (`oklch(0.85 0.15 85)`)
- **Reconciliation**: Soft pink (`oklch(0.80 0.12 340)`)

**Typography**:
- Headings: **Montserrat** (geometric, modern, confident)
- Body: **Inter** (humanist, readable, friendly)
- Timestamps/mono: **JetBrains Mono**

**Animation philosophy**: Spring physics (`cubic-bezier(0.34, 1.56, 0.64, 1)`). Micro-interactions 150ms, message arrivals 250ms, ritual effects 400ms. Everything should feel physical — like objects with weight. `motion/react` (Framer Motion) throughout.

**Dual aesthetic modes**:
- **Ornate** (default): Full animations, glow effects, spring physics, acoustic fields
- **Minimal**: Static, instant, no decoration — for focus or accessibility

**References**: macOS Messages (glass + blue), Telegram (speed + cleanliness), Yahoo! Messenger (emotional warmth + ritual sense)

**Anti-references**:
- ❌ Slack — corporate, cold, transactional. No utility-first sterility.
- ❌ LINE — chaotic sticker culture, ad-cluttered, visually noisy. Fix the clutter, keep the warmth.
- ❌ Pure minimalism — white backgrounds, no animation, no personality. Restraint ≠ emptiness.

---

### Design Principles

1. **Ritual over Transaction** — Every send, receive, and interaction is a moment, not a data packet. Design for the emotional memory, not the function. The `InteractionEvent` pattern (Sonic Ping, Apology, Celebration) is the heart of this principle.

2. **Warm Precision** — Warmth and craft are not opposites. Spacing must be exact, timing must feel right, glassmorphism must be curated. Imprecise warmth is just sloppiness. Precise warmth is Ping.

3. **Clean ≠ Stripped** — Visual cleanliness is achieved through *hierarchy and restraint*, not removal. Every element earns its place. No ads, no forced notifications, no decorative noise — but plenty of soul.

4. **Depth through Layering** — Color, motion, and spatial relationships communicate the *state of a relationship*, not just its messages. A conversation with 0 unread feels different from one with a Sonic Ping. The glass layers, ritual colors, and animations carry this meaning.

5. **Both Platforms, One Soul** — Web glassmorphism and Mobile NativeWind should feel like siblings, not strangers. Spring physics on mobile should feel as physical as on desktop. The same design token drives both.

---

### Current Priority

**Polish and refinement** — The architecture and feature set are solid. The next level is the *feel*: consistent spacing, perfectly-timed micro-interactions, hover states with no layout shift, ritual animations that make users stop and notice. The gap between "works" and "delights" is in the details.

---

### Technical Constraints

- **Colors**: OKLCH only (`oklch(L C H)` or `oklch(L C H / alpha)`). CSS relative color syntax (`oklch(from var(--token) l c h / 0.5)`).
- **CSS variables**: All design tokens live in `@shared/design-tokens`. Web: `var(--token-name)` from CSS. Mobile: TypeScript imports.
- **Icons**: Lucide React only. No emoji as icons.
- **Animations**: `motion/react` (Framer Motion). Respect `prefers-reduced-motion` and `[data-aesthetic="minimal"]`.
- **Components**: Use `.glass-button`, `.glass-card`, `.glass-input` classes. Extend via `--interaction-color` CSS variable pattern (see `InteractionEvent`).
- **No hardcoded colors** in components. No inline styles for theming. CSS custom properties only.
- **WCAG AAA** target: contrast > 7:1, keyboard navigable, full ARIA labels, `role="status"` for live regions.
- **Web**: TanStack Start + React 19 + Tailwind CSS v4. **Mobile**: React Native 0.81 + Expo 54 + NativeWind.
